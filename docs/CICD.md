# CI/CD Guide

This document describes the CI/CD pipeline for SkillsJars Spring AI Example.

## Pipeline Overview

```
Pull Request Opened
    │
    ▼
┌─────────────────────────────────┐
│    PR VALIDATION                │
│    - PR title format check      │
│    - Compilation verification   │
│    - Dependency check           │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│    CI BUILD                     │
│    - Build and Test             │
│    - Security Audit (OWASP)     │
│    - Project Validation         │
└─────────────────────────────────┘
    │
    ▼
  Ready for Merge
    │
    ▼
┌─────────────────────────────────┐
│    SNAPSHOT PUBLISH             │
│    (on merge to main)           │
└─────────────────────────────────┘
```

## Workflows

### 1. CI Build (`ci.yml`)

Runs on every push to `main` and pull requests.

**Jobs:**
- **build** - Compiles code and runs tests
- **security-audit** - Runs OWASP dependency check
- **validate** - Validates project structure

**Triggers:**
- Push to `main`
- Pull request to `main`

### 2. PR Validation (`pr-validation.yml`)

Runs on pull requests to ensure code quality before merge.

**Jobs:**
- **validate-pr** - Validates PR title follows conventions
- **verify-compilation** - Ensures code compiles
- **check-dependencies** - Checks for dependency updates

**Triggers:**
- Pull request opened, synchronized, reopened

### 3. Publish Snapshot (`publish-snapshot.yml`)

Automatically publishes SNAPSHOT versions to Maven Central.

**Triggers:**
- Push to `main`
- Manual workflow dispatch

### 4. Release (`release.yml`)

Manual workflow for releasing to Maven Central.

**Inputs:**
- `version` - Release version (e.g., 0.1.0)
- `createTag` - Whether to create Git tag (default: true)

**Steps:**
1. Update version in pom.xml
2. Build and verify
3. Publish to Maven Central with GPG signing
4. Create Git tag and GitHub release
5. Bump to next SNAPSHOT version

**Triggers:**
- Manual workflow dispatch

## Quality Gates

| Gate | Tool | Purpose |
|------|------|---------|
| Compilation | Maven | Ensures code compiles |
| Tests | Maven Surefire | Validates functionality |
| Code Style | Checkstyle | Enforces Google Java Style |
| Dependencies | Enforcer | Version requirements |
| Security | Trivy | Vulnerable dependencies (fast scan) |

> **Note:** Trivy replaces OWASP Dependency Check for faster scan times. OWASP is still available locally: `./mvnw dependency-check:check`

## Required Secrets

Configure these in GitHub Repository Settings → Secrets and variables → Actions:

| Secret | Description | Required For |
|--------|-------------|--------------|
| `MAVEN_USERNAME` | Sonatype Central username | Snapshot/Release |
| `MAVEN_PASSWORD` | Sonatype Central password | Snapshot/Release |
| `GPG_SECRET_KEY` | GPG private key | Release only |
| `GPG_PASSPHRASE` | GPG private key passphrase | Release only |
| `GITHUB_TOKEN` | Auto-generated by GitHub | PR validation |

## PR Title Conventions

PR titles must follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add new PDF export skill
fix: resolve null pointer in logging advisor
docs: update README with security notes
refactor: simplify ChatClient configuration
style: format code according to style guide
test: add unit tests for MyLoggingAdvisor
chore: update dependency versions
build: modify build configuration
ci: update CI workflow
perf: improve performance
revert: revert previous commit
```

## Manual Operations

### Trigger Snapshot Release

1. Go to Actions → Publish Snapshot
2. Click "Run workflow"
3. Select branch (default: main)
4. Click "Run workflow"

### Create New Release

1. Go to Actions → Release
2. Click "Run workflow"
3. Enter version number (e.g., `0.1.0`)
4. Check "Create Git tag after release"
5. Click "Run workflow"

### Rollback a Failed Release

If a release fails partially:

1. Check Maven Central for published artifacts
2. Contact Sonatype support to remove artifacts if necessary
3. Clean up Git tags locally and remotely:
   ```bash
   git tag -d v0.1.0
   git push origin :refs/tags/v0.1.0
   ```
4. Fix the issue and retry

## Troubleshooting

### Build Fails with "Out of Memory"

Increase Maven heap size:
```bash
export MAVEN_OPTS="-Xmx2048m"
```

Or in CI, modify `ci.yml`:
```yaml
- name: Build with Maven
  run: ./mvnw clean verify -B
  env:
    MAVEN_OPTS: "-Xmx2048m"
```

### Dependency Check Fails (Trivy)

1. View the GitHub Security tab for vulnerability details
2. Update the vulnerable dependency
3. If false positive, add to `.trivyignore` in project root

Example `.trivyignore`:
```
# Accept the risk of CVE-2021-XXXX
CVE-2021-XXXX
```

### OWASP Dependency Check (Local Only)

If running OWASP locally for deeper analysis:

1. Review the report in `target/dependency-check-report.html`
2. Update vulnerable dependencies
3. If false positive, add suppression in `dependency-check-suppressions.xml`

### Checkstyle Fails

Run locally to see details:
```bash
./mvnw checkstyle:check -X
```

Fix violations or update checkstyle configuration if needed.

### GPG Signing Fails

Ensure GPG secrets are properly configured:
1. GPG_SECRET_KEY should be the armored private key
2. GPG_PASSPHRASE should match the key passphrase

Generate armored key:
```bash
gpg --armor --export-secret-keys your-key-id
```

## Optimization

Current pipeline optimizations:

- ✅ Maven dependency caching
- ✅ Parallel job execution
- ✅ Fail-fast enabled
- ✅ Artifact retention policies

For slow pipelines, consider:
- Running security audit on schedule only
- Skipping unrelated tests based on changed files
- Using larger GitHub runners
